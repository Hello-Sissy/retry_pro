> **与 `01_错题库` 分工**
>
> | 库 | 记什么 |
> |:---|:---|
> | **01_错题库** | 按**题**归档；§4 保留单题复盘轨迹 |
> | **本目录** | 按**知识点**合并深度规则；**`组卷要求` 唯一出处** |

错题过程中若某条知识点已挖透，收入对应**子类**专题；原子题 YAML 只写 `知识专题` 链接，**不写 `组卷备注`**。

## 命名与 ID

- **文件名**：`{知识点名称}_{YYYYMMDD}.md`（`YYYYMMDD` = 建档日）
- **专题 id**：`TOPIC-{科目代码}-{子类缩写}-{YYYYMMDD}-{序号}`（如 `TOPIC-ENG-GRAM-20260608-01`）
- **原子题链接**：`知识专题: "{basename}"`（无 `.md`）；解析路径 `05_不熟练知识专区/{科目}/{子类}/{basename}.md`

## 关联维护

凡错题 YAML 写入或继承 `知识专题`，或批改新建 `#变式题` 档，**必须**同步专题：

| 专题字段 | 内容 |
|:---|:---|
| **`关联错题`** | 该专题下全部挂链 `id`（母题 + 变式错题档） |
| **`关联变式`** | 其中 `#变式题` / `母题` 非空的子集 |
| **`## 关联错题` 表** | 与上同步增行 |
| **子类 `README.md`** | 在练索引的关联错题数等 |

专题 `mastery` 取 `关联错题` 各档**最低值**（`status: 在练` 时）。

**已掌握后新增关联档** → 专题**打回在练**（见下）；不批量改动其他已毕业错题。

---

## 毕业与打回

**仅母亲判断**；系统不在批改后自动提示或触发毕业。

### 毕业

口令：`what与how 已掌握` / `毕业 what与how`

**有关联错题尚未毕业**（`mastery` 非 4 或不在 `已毕业/`）时：**先列出关联错题表反问，不直接改状态**；你回 **`全部毕业`** 后再执行落盘。

**关联错题均已毕业**时：直接落盘，无需二次确认。

落盘动作：

1. 专题 → `status: 已掌握`，`mastery: 4`，移入 `…/已掌握/`。
2. 凡 `知识专题` 指向该 basename 的错题 → `mastery: 4`，移入 `01_错题库/…/已毕业/`。
3. 更新子类 README、`## 变更记录`。

### 打回

专题已 `已掌握` 后，因录入 / 挂链 / 变式建档**新增**关联错题 →

1. 专题 → `status: 在练`，`mastery` 取关联最低值，移回在练目录。
2. 既有已毕业错题档**不动**；新档按正常规则留在在练。

## 组卷读取链

1. `mapping` 错题 YAML 有 `知识专题` → 打开专题 md（`status: 已掌握` 的专题**不参与**默认 8+2 与 `组 … 专题`）
2. 读 **`组卷要求`** + **`## 规则精讲`**
3. 设计变式/干扰项/概念热身；确认清单附 **`组卷要求` 摘要**
4. 同卷多题挂同一专题 → 要求只读一次

无 `知识专题` 的题 → `variants` + 默认变式引擎。

---

## 操作口令

| 口令 | 动作 |
|:---|:---|
| `建知识专题 what与how` | 新建专题 + 双向关联 + 关联维护 |
| `同步 what与how 专题` | 从关联错题 §4 更新规则精讲 |
| `组 what与how 专题` | 专题组卷（仅 `status: 在练`） |
| `what与how 已掌握` / `毕业 what与how` | 专题毕业（有未毕业关联错题时先反问） |
| `全部毕业` | 确认后专题 + 关联错题一并毕业 |

权威规则：工作空间根目录 `.cursorrules` **§3.7**。
