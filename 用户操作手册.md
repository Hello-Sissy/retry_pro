# 用户操作手册 · MistakeMaster 错题中台

> 仓库：<https://github.com/Hello-Sissy/retry_pro>  
> 可 fork 的模板：AI 规则 + 目录结构 + 少量样例题，不含家庭隐私数据。  
> **环境**：Cursor（或支持 `.cursorrules` 的 IDE）+ Python 3 + Git。打开时选**根目录**（含 `.cursorrules` 的那层），不要只打开 `错题记录/`。

```bash
git clone https://github.com/Hello-Sissy/retry_pro.git
# 用 Cursor：文件 → 打开文件夹 → 选 MistakeMaster
```

## 1. 这个系统可以做什么？

> 一句话：**用 AI 替代手工错题本**——拍截图录入、自动出卷、批改回写、掌握度追踪，全靠对话完成，数据留在本地 Markdown 文件。

### 核心功能

| 功能 | 你说什么 | AI 做什么 |
|:---|:---|:---|
| **录入错题** | 发截图 | OCR → 诊断错因 → 反问确认 → 写 `.md` 入库 |
| **录入默写错词** | `伶俐默写错了` | 只更新词表一行，不建错题档案 |
| **录入英语短语/词性** | `plenty of 默写错了` | 更新对应英语默写词表 |
| **组卷** | `组` / `组数学` / `组英语` | 从题库加权抽题 → 出确认清单 → 你回 `组` → 落盘 `.md` + `.pdf` |
| **批改** | `批改 exam_260603_01：1对，2错` | 升降掌握度、写入错误次数、自动移入已毕业 |
| **不熟练知识专项** | `建知识专题 what与how感叹句` | 建知识点文件，关联相关错题，可单独出专项卷 |
| **默写卷** | `出语文默写卷` | 从词表抽题出卷，批改后更新词表状态 |

### 全部口令速查

**录入**

| 口令 | 含义 |
|:---|:---|
| `按推荐` | 接受 AI 推荐的标签/错因/掌握度，整题入库 |
| `8题：审题` | 只改第 8 题的错因 |
| `改略懂` | 只改掌握度 |
| `来源：杨浦区2025一模` | 补充题目来源 |

**组卷 & 批改**

| 口令 | 含义 |
|:---|:---|
| `组` / `组数学` / `组英语` | 默认 10 题（8 在练 + 2 复检），出确认清单后回 `组` 落盘 |
| `含超纲` | 数学组卷时纳入超纲题 |
| `批改 exam_YYMMDD_nn：1对，2错，1.1对` | 按题号批改，含变式（1.1） |
| `按推荐组卷` | 确认清单后直接落盘 |

**知识专题**

| 口令 | 含义 |
|:---|:---|
| `建知识专题 [知识点名]` | 新建专题 + 关联错题 |
| `同步 [知识点名] 专题` | 从关联错题更新专题规则 |
| `组 [知识点名] 专题` | 只出该专题的题 |
| `[知识点名] 已掌握` | 专题毕业（需确认关联错题） |

**默写**

| 口令 | 含义 |
|:---|:---|
| `XX默写错了` | 更新语文默写词表 |
| `出语文默写卷` | 从词表出默写卷 |
| `批改 dictation_YYMMDD_nn：1对 2错` | 批改默写卷，更新词表 |

## 2. 五分钟跑通一条链

在 Cursor 对话框依次试（样例数据已内置）：

| 步骤 | 你对 AI 说 | 预期结果 |
|:---|:---|:---|
| ① 看样例 | 「打开 `eng_20260601_05_advice不可数.md` 讲结构」 | AI 讲解 YAML + 四段正文的含义 |
| ② 组卷 | **`组英语`** | 先出确认清单 |
| ③ 确认 | **`组`** | 落盘 `exam_*.md` + `exam_*.pdf` |
| ④ 批改 | **`批改 exam_260603_01：1对，2错`** | 更新错题 `mastery` / `错误次数` |

手动补跑 PDF（在 `错题记录/` 下）：

```bash
python3 scripts/md_to_exam_pdf.py "02_自定义组卷/已完成/exam_260603_01.md"
```

打印时：英语卷只给孩子「**第一部分 + 答题卡**」，第三部分标准答案和深度分析撕掉/不打印。

## 3. 深度使用

### 3.1 错题掌握度四档

| 值 | 含义 | 录入默认 | 毕业条件 |
|:---:|:---|:---|:---|
| 1 | 很不懂 | M1/C1/E1 错因时 | — |
| 2 | 略懂 | 思路卡壳类错因 | — |
| 3 | 懂但粗心 | 计算/审题/拼写错因 | — |
| 4 | 会 | **不可录入时直接给**；只能由批改升至 | 移入 `已毕业/` |

组卷时 `mastery < 4` 的题为**在练槽**，`mastery = 4` 的题随机抽 2 道进**复检槽**（题号旁带「〔复检〕」）。

### 3.2 变式题机制

每道数学错题录入时，AI 自动写入 2 条变式策略（`variants` 字段）。组卷时：

- 掌握度 1–2：出 **2 道变式**（L1 换问法 + L2 加条件）
- 掌握度 3：出 **1 道变式**，连续做对 2 次自动升档（L1→L2→L3）
- **L 档封顶**：K1/K2/K4/K5 封顶 L3，K3（应用题）封顶 L4；封顶后连续对 2 次转为毕业考核

变式做错时自动新建独立错题档（`#变式题` 标签），不修改母题。

### 3.3 知识专题（不熟练知识专区）

一道错题里写不完的**系统性弱点**（如"感叹句 what/how 分不清"），可以建一个知识专题把相关错题串起来：

```
建知识专题 what与how感叹句
```

AI 会建 `05_不熟练知识专区/英语/语法/what与how感叹句_20260608.md`，写规则精讲 + 组卷要求，并双向关联所有涉及该知识点的错题。之后用 `组 what与how感叹句 专题` 可单独出该知识点的卷。

### 3.4 英语默写词表（与错题库独立）

| 词表 | 触发 | 文件 |
|:---|:---|:---|
| 专有名词 | 国名/人名/地名不认识 | `03_英语默写/专有名词词表.md` |
| 短语 | 中译英短语写错 | `03_英语默写/短语词表.md` |
| 句子 | 中译英整句写错 | `03_英语默写/句子词表.md` |
| 词性转换 | 原词→目标词性写错 | `03_英语默写/词性转换词表.md` |

连续写对 **2 次** → 状态自动改为 `已掌握`。这四张词表**不算错题**，不会被组入 `exam_*` 卷。

### 3.5 语文默写卷（`dictation_*`）

字词/成语默写错了只改词表，不建错题档案。词表积累后：

```
出语文默写卷
```

卷面只印拼音，不印汉字；批改后词表自动更新连续对次数。

## 4. 目录结构

```
MistakeMaster/
├── .cursorrules                  ← AI 总规程（必须加载根目录才生效）
├── 用户操作手册.md               ← 本页
└── 错题记录/
    ├── .cursorrules              ← 子模块组卷细则
    ├── 01_错题库/                ← 每题一个 .md
    │   ├── 数学/K1–K5/           ← 在练；已做对的在 已毕业/ 子目录
    │   ├── 语文/C1–C4/
    │   └── 英语/E1–E4/
    ├── 02_自定义组卷/
    │   ├── 未完成/               ← exam_YYMMDD_nn.md + .pdf
    │   ├── 已完成/               ← 批改完成后 md+pdf 移入
    │   ├── _exam_template.html   ← PDF 排版模板
    │   └── _snippets/            ← 几何题 SVG 片段
    ├── 03_英语默写/              ← 四张词表（不算错题）
    ├── 04_语文默写清单/          ← 默写词表 + dictation_* 卷
    ├── 05_不熟练知识专区/        ← 知识专题（英语/语法/ 等）
    ├── assets/images/            ← 原题截图
    ├── scripts/
    │   ├── md_to_exam_pdf.py     ← Markdown → PDF
    │   └── pick_exam_weighted.py ← 加权抽题逻辑
    └── 错题整理架构.md           ← 模块码/错因码/标签字典
```

## 6. 定制为自己孩子用

1. **Fork** 本仓库到你的 Gitee/GitHub（建议**私有**，避免真实错题泄露）。
2. 把 `01_错题库/` 下样例题全部删掉，从自家试卷开始录入。
3. 改 `错题记录/错题整理架构.md` 里的年级锚点和模块说明（可选）。
4. 根目录 `.cursorrules` 较长，一般只需改「考纲锚点（年级/学段）」「默认组卷题量」「城市/升学政策」几处；组卷细则在 `错题记录/.cursorrules` 修改。
5. 默认题量 10（8+2）；改成 8 题只需组卷时说 `组8题`，无需改规则。

## 7. 常见问题

**Q：打开后 AI 不按规则走？**  
A：必须打开 **MistakeMaster 根目录**（含 `.cursorrules` 的那层），不能只打开 `错题记录/`。

**Q：图片在卷子里裂了/不显示？**  
A：录入时把原题截图放进 `assets/images/`；错题 md 里路径写 `../../../assets/images/图片名`。

**Q：仓库里为什么没有 PDF？**  
A：有意不提交。克隆后本地跑 `md_to_exam_pdf.py` 生成，避免二进制文件污染 Git 历史。

**Q：变式答案写在哪里？**  
A：英语新卷用答题卡模式——变式答案只写在**第二部分答题卡**，题卷不作答。数学/语文 legacy 卷的变式答案写在卷末 `## 标准答案`。

**Q：录入了字词默写错误，为什么没建错题档案？**  
A：字词/成语默写只改 `04_语文默写清单/默写词表.md`，不走错题库，这是刻意设计——默写是记忆任务，和知识理解类错题分开管理。

**Q：知识专题和错题有什么区别？**  
A：错题是**一道题的档案**；知识专题是**一个知识点的规则书**，把多道相关错题串起来，写清楚怎么出变式、怎么带练，是"错题集"之上的一层。

**Q：和市面上的错题 App 比？**  
A：数据在本地 Markdown + Git，规则完全可改；AI 做 OCR / 诊断 / 组卷 / 批改回写，不依赖任何订阅服务。
